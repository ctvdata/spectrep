Instrucciones para realizar pruebas en gamma:

1.Manejo de contenedores:
    1.1 Conectarse al servidor mediante ssh con las credenciales entregadas.
    1.2 Crear un contenedor a partir de la imagen mcrespo/pan:1.0. Esto solo se realiza la primera vez mediante el comando:
        docker run -ti -v <volumen local>:<volumen dentro del contenedor> --name <nombre del contenedor> mcrespo/pan:1.0
    1.3 Para iniciar o detener el contenedor basta con usar los comandos docker start/stop <nombre del contenedor>
    1.4 Para volver a entrar al contenedor una vez fuera de el se usa el comando:
        docker exec -ti <nombre del contenedor> bash
    1.5 para listar los contenedores activos se usa el comando:
        docker ps (lista los contenedores activos)
        docker ps -a (lista todos los contenedores, activos e inactivos)
    
2. Realizacion de pruebas:
    2.1 Para realizar pruebas se recomienda hacer uso de tmux para evitar perder la terminal en caso de una desconexión. Para ello desde la terminal de linux usamos el siguiente comando:
        tmux new-session -s <nombre de la sesión>
    2.2 Para salir de la terminal multiplexada usamos el siguiente comando:
        ctrl + B soltamos y presionamos D
    2.3 Para listas las termiales multiplexadas activas usamos el siguiente comando:
        tmux list-sessions
    2.4 Para entrar nuevamente a la terminal multiplexada usamos el siguiente comando:
        tmux attach-session -t <nombre de la sesión>
    2.5 Los programas y los datos deben ser copiados al volumen local que indicaron en la creación del contenedor del punto 1.2 para que este pueda verlos.
    2.6 Dentro del contenedor pueden ejecutar los archivos mediante los comandos típicos de python.
